## 一、项目目标演进

项目最初目标是完成一个双机协同的课堂图像远程监测系统：前端电脑采集摄像头画面，通过自定义 TCP 协议发送到后端电脑；后端电脑接收画面、运行 YOLO 模型进行识别，并在出现异常行为时保存报警记录。

随着实验推进，系统目标逐步从“单一异常检测”扩展为“多行为识别、模型对比、现场数据适配和大模型辅助分类”。最终形成了两条演示链路：

- 正式双机链路：Qt 前端发送端 + 自定义 `NSGD` TCP 帧包 + Qt 后端分析端。
- 单机展示链路：浏览器 Web 控制台，用于图片、视频和摄像头快速演示。

在模型方案上，项目经历了三次主要转向：

1. 直接使用课堂行为 YOLO 六分类模型。
2. 使用新数据集继续训练六分类 YOLO，并对不同权重进行验证。
3. 重构为“YOLO 只做人形检测 + 大模型对编号人体图进行行为分类”的两阶段方案，以提高人体框稳定性，并保留六分类 YOLO 作为回退方案。

## 三、重要功能变更记录

### 1. 通信协议

项目使用自定义 `NSGD` TCP 帧包协议，保留课程网络设计特色。前端发送端采集摄像头画面并编码成 JPEG，按照自定义包头发送；后端监听端负责收包、拆包、解码和推理。

主要实现：

- `src/common/protocol.py`
- `src/common/image_codec.py`
- `src/frontend/gui_client.py`
- `src/backend/gui_app.py`

### 2. 后端识别模式

后端目前保留两种识别模式：

- `人体YOLO+大模型`：当前默认方案，YOLO 检测人体，大模型分类行为。
- `六类YOLO`：历史回退方案，直接使用训练好的六类 YOLO 权重检测课堂行为。

相关模型：

- `yolov8s.pt`：当前默认人体检测模型。
- `models/merged_classroom_6cls_v2_img960_e50_2026-06-13_best.pt`：六类 YOLO 回退模型。
- `models/classroom_behaviour_6cls_2024-10-02_baseline.pt`：原始基线模型。
- `models/student_behaviour_v6_6cls_img960_e50_2026-06-12_best.pt`：Student Behaviour Detection v6 训练模型。

### 3. 大模型分类

大模型输入不是整张原图，而是 YOLO 检出的人体框拼图：

1. YOLO 检测课堂画面中的 `person`。
2. 后端按人体框外扩裁剪。
3. 将每个人体裁剪图拼成编号大图。
4. 上传给大模型，要求返回每个编号对应的六类行为。
5. 根据编号把行为结果回填到原图人体框上。

可选大模型：

- GPT-5.5，走 OpenAI 兼容接口。
- 千问 qwen3.6-flash，走 DashScope 接口。

后端界面已支持运行时选择大模型和目标上限。

### 4. 行为类别展示

底层六类标签：

- `Hand-raise`
- `Reading`
- `Writing`
- `Useing-Phone`
- `Head-down`
- `Sleeping`

系统展示口径：

- `Reading` 和 `Writing` 合并显示为“学习”。
- `Useing-Phone`、`Head-down`、`Sleeping` 作为异常行为。
- 报警时只给异常目标标红，不再全员标红。

### 5. 本地测试与演示

后端 GUI 支持三类演示：

- 双机监听：接收前端发送端的摄像头画面。
- 图片测试：直接选择本地课堂图片进行检测。
- 视频测试：直接选择本地课堂视频逐帧检测。

浏览器 Web 控制台支持：

- 摄像头输入。
- 图片输入。
- 视频输入。
- YOLO 实时分析。
- 大模型分类展示。
- 日志与报警记录。

### 3. 双机联调

双机联调流程：

1. 后端运行 `START_BACKEND_GUI.ps1`，监听 `0.0.0.0:5001`。
2. 前端运行 `START_FRONTEND_GUI.ps1`，填写后端 IP 和端口。
3. 前端采集摄像头画面，通过 TCP 发送。
4. 后端接收并显示检测结果。
5. 异常行为持续超过阈值时保存截图和 CSV 记录。

该流程保留了课程项目的网络设计核心。
