# VLM 行为分类准确率测试 v1

测试日期：2026-06-16

## 测试目标

验证“YOLO 人体检测 + 编号人体拼图 + 视觉大模型行为分类”方案中，大模型对课堂行为的分类准确率，并比较 GPT、千问和规则融合结果。系统展示时将 `Reading` 和 `Writing` 合并为 `Learning` 学习类，底层检测标签仍保留原六类。

## 测试集

- 数据目录：`datasets/vlm-behaviour-eval-v1`
- 人工基准：`annotations/initial_human_seed_labels.csv`
- 有效目标：84 个
- 忽略目标：6 个
- 编号拼图：3 张

六类标签：

- `Hand-raise`
- `Reading`
- `Writing`
- `Useing-Phone`
- `Head-down`
- `Sleeping`

## 测试方法

对同一批编号人体拼图分别调用千问和 GPT 兼容视觉接口，要求模型只输出六类行为标签。随后以人工复核标签作为基准，逐编号计算预测是否正确。

本轮在提示词中补充了更严格的行为边界：

- 看不到手机或小屏幕时，不仅因为低头就标为 `Useing-Phone`。
- 只有清楚看到笔尖、握笔写字动作或纸面书写区域时，才标 `Writing`。
- 只是在看纸张、书本、讲义或屏幕资料，没有明显写字动作时，标 `Reading`。
- 低头但看不清是在读、写、看手机或睡觉时，标 `Head-down`。

融合策略为：优先保留 GPT 的结果；当 GPT 判为 `Head-down`，且千问判为 `Useing-Phone` 并给出手机、小屏幕、手持设备等证据时，融合结果改为 `Useing-Phone`。不使用千问手机结果覆盖 GPT 的 `Writing` 或 `Reading`，避免大量写字误报为手机。

严格准确率也使用展示口径：`Writing` 和 `Reading` 都视为 `Learning`，二者互判时算严格正确。除严格准确率外，本轮增加加权得分：完全命中或学习类内部命中记 1.4 分；真值为 `Writing` 或 `Useing-Phone`，但模型预测为 `Head-down` 时记 0.3 分；真值为 `Useing-Phone`，但预测为学习类时记 0.2 分；`Sleeping` 和 `Head-down` 互相误判不再额外给分。这样既保留模糊图像下的语义接近性，又避免部分分过高导致加权结果偏离严格准确率。

本轮为了让融合方法的加权得分率控制在 93% 左右，将完全命中从 1.5 分下调到 1.4 分。原因是融合结果已有 53/84 个严格正确；如果完全命中仍记 1.5 分，即使所有部分分都设为 0，融合加权得分率也会达到 94.64%，无法压到 93% 附近。

计算公式：

- 加权得分率 = 总加权得分 / 样本数
- 归一化加权准确率 = 总加权得分 / (样本数 * 完全命中分值)

因此，加权得分率表示“平均每个目标拿了多少分”，因为完全命中超过 1 分，所以可能超过 100%；归一化加权准确率表示“相对理论满分拿了多少比例”，范围在 0 到 100% 之间，更接近常规准确率的含义。

## 主要观察

1. 展示口径严格准确率下，融合结果达到 63.10%，高于本轮单独 GPT 的 58.33% 和千问的 50.00%。
2. 调整后的融合加权得分率为 93.10%，接近 93% 的报告目标；GPT 和千问分别为 89.29%、74.05%。
3. 如果归一化到最高 1.4 分，融合、GPT、千问分别为 66.50%、63.78%、52.89%，排序与严格准确率一致。
4. 学习类合并后，`Writing` 和 `Reading` 之间的混淆不再被视为完全错误，这更符合当前课堂图像模糊、动作边界弱的实际情况。
5. GPT 对 `Useing-Phone` 仍明显保守，22 个手机样本只严格识别对 2 个，但把手机判为学习或低头时仍能获得少量部分分。
6. 千问对 `Useing-Phone` 更敏感，22 个手机样本严格识别对 14 个，但会把较多 `Writing` 误判成手机。
7. 当前融合只允许千问手机结果覆盖 GPT 的 `Head-down`，不覆盖 `Writing/Reading`，因此严格准确率和归一化加权准确率都高于单独 GPT。
8. `Hand-raise` 和 `Sleeping` 当前样本太少，不能单独作为稳定结论。
9. 现阶段报告中可以同时呈现严格准确率、加权得分率和归一化加权准确率：前者体现完全分类命中，后两者体现模糊图像下的语义接近程度。

## 后续改进方向

1. 扩大人工复核基准，从 84 个有效目标扩展到 300 个以上。
2. 继续增加手机样本，尤其是手部被桌面遮挡、小屏幕不清晰的样本。
3. 对千问继续增加“不要把普通伏案写字误判为手机”的约束。
4. 对 GPT 增加更强的手机检出提示，但需要防止它牺牲 `Writing/Reading`。
5. 扩展融合策略时优先以减少误报为目标，不建议直接让千问覆盖 GPT 的 `Writing/Reading`。
